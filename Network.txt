host - սարք, որը կարող է ցանցով ինֆորմացիա ուղղարկել մի վերջնակետից մյուս վերջնակետը։
ip (internet protocol) - մի հոսթից մյուսին ինֆորմացիա ուղղարկելու համար, անհրաժեշտ է հոսթը ունենա ինդենտիֆիկացվելու մեխանիզմ։
local ip - չի կրկնվում միայն տվյալ լոկալ ցանցի ներսում։
global ip - չի կրկնվում ամբողջ աշխարհում, միանշանակ կարելի է իդենտիֆիկացնել հոսթը այս ip֊ով։
internet - համաշխարհային ցանց
ipv4 - 4 բայթանոց ip, սկզբում միայն այս տիպի ip֊ներն էին։ Սակայն host֊երի շատանալուն զուգահեռ, 4 բայթանոց ip֊ների դիապազոնը սկսվեց սպառվել։ 0-255 միջակայքում 4 թիվ է։ Օրինակ՝ 192.168.1.20
ipv6 - 6 բայթանոց ip, գործնականում երբեք չի սպառվի։ Ip-ի մի քանի բայթով որոշվում է ցանցը, մի քանիսով տվյալ ցանցի մեջ կոնկրետ հոսթը։ Ցանցի ու հոսթի բայթերի քանակը ֆիքսված չի, դա կարող է տատանվել անգամ բիթերի մակարդակով։ ipv6-ը թիվը պահում է 16-ական համակարգով։

OSI  7 layers - ստանդարտ է որով մշակվել է մակարդակներ, որոնցով անցնում է տվյալը ցանցով ուղղարկվելուց։
    1. Application  ֊ ծրագրային մակարդակն է, որով տվյալը ուղղարկվում է։ 
    2. Presentation ֊ ուղղարկվող ֆայլը կարող է առանց կորստի խտացվել(rar, jpg, mpeg ...), մասնատվում է և համարակալվում է։ Ինֆորմացիան տեղ է հասնում տարբեր ճանապարհներով, վերջնակետում հավաքելուց օգտագործվում է համարակալումը։
    3. Session ֊ նախանշում է կապի սկիզբն ու ավարտը։
    4. Transport ֊ այս մակարդակում որոշվում են պորտերը։ Տվյալի վրա ավելանում է ստացողի (destination) և  ուղղարկողի (source) պորտերի համարները, որոնք ընտրվել են random տիրույթից։
    5. Network ֊ այս մակարդակում ավելանում են source և destination ip֊ները։ Այս մակարդակում ինֆորմացիան արդեն կոչվում է պակետ (packet):
    6. Data link ֊ այս մակարդակում ավելանում են source և destination macaddres֊ները։ Այս մակառդակի վրա կոչվում է frame:
    7. Physical ֊ ֆիզիկական մակարդակում ինֆորմացան վերածվում է բարձր ու ցածր մակարդակների լարման ու տեղափոխվում է մյուս հոսթ։ Ընդունող հոսթում այս մակարդակները հակարակ հերթականությաբ անցնելով, ավելորդ ինֆորմացիան գցելով, (macaddres, ip, port), արխիվից կամ խտացված վիճակից հանելով, ստացվում է ինֆորմացիան իր օրիգինալ տեսքով։

port - ip֊ին նման է հասցեի, նամակ ուղղարկելիս պետք է նշվի հասցեն և թե տվյալ հասցեում ում է ուղղված նամակը։ Դրա դերը տանում է port֊ը։ Սերվերների վրա տեղադրված եմ լինում տարբեր ծառայություններ, ամեն մի ծառայության դիմելիս, պիտի դիմենք իր պորտով, որովհետև ամեն մի ծարայություն ունի իր պորտի համարը։ 
Վեբ ծարայությանը դիմելիս կատարում ենք http, https  հարցումներ։
http ֊ port֊ի համարը 80
https - port֊ի համարը 8080, s֊ը նշանակում է secure (անվտանգ), այսինքն միջանկյալ գաղտնագրում է տեղի ունենում։
FTP service (file transfer protocol) port֊ի համարը - 21, ֆայլ ուղղարկել, ստանալու ծառայություն։
SMTP service (simple mail transfer protocol) port֊ի համարը - 25
Երբ մենք վեբ կայք ենք բացում, տակից աշխատում են տարբեր ծարայություններ։

ports range
0 - 1023 - աշխատում հայտնի ծարայությունների գրանցված պորտերն են։ Օրինակ՝ SSH (secure shell) գաղտնագրված է- port֊ի համարը 22, telnet - հեռակա կարավարման ծրագիր է, որը գաղտնագրում չի անում, օգտագործվում է լոկալ ցանցի մեջ․շ, port֊ի համարը 23։ Օպերացիոն համակարգում հիմանականում արդեն ներդրված են լինում այս սահմանի պորտեի վրա աշխատող ծրագրերը։
1024 - 49151 ֊ կոմերցիոն տարբեր ծրագրերինն է
49152 - 65535 ֊ random տիրույթ է կոչվում, ինչ֊որ սերվիսի դիմելուց հետադարձ հասցեն թողնելիս, ip֊ին մեր ip֊ին է, իսկ որպես պորտ կարող ենք նշել այս տիրույթից որևէ պորտ։ Պատասխանը գալիս է նշված պորտին։ Դա կատարվում է ատոմատ, բայց սոկետների հետ աշխատելիս կարող ենք ձերքով տալ։

macaddres - host֊ի ինտերնետի հետ կապ հաստատելու սարքի անունը LAN քարտ է։ Դրա վրա ֆիզիկապես դաջված է մակ հասցեն, որը փորձում են, որ չկրկնվի։ Օպերացիոն համակրգը միանալիս կարդում է այդ դաջված թվերը, և ներբեռնում։ Արդեն ՕՍ֊ի ներսից կարելի է փոխել։ Մակ հասցեն միայն լոկալ ցանցում կողմնորոշվելու համար է։ Այն լոկալ ցանցից դուրս չի երևում։ Մակ հասցեն նաև օգտագործվում է ցանցին առաջին անգամ միանալու պահին ip  ստանալիս, host֊ը իդենտիֆիկացվում է ըստ macaddres֊ի։

Մի հոսթից մյուս հոսթին ինֆորմացիան ուղիղ չի գնում, այն անցնում է միջանկյալ սարքերի միջով։

swich - աշխատում է լոկալ ցանցում, միայն մակ հասցեների հետ, իր միջոցով կարող է ինֆորմացիան փոխանցվել դեպի router, իր միջով ուղղորդելիս ավելացնում է մյուս ցանցի մակ հասցեն, հետո հանում։ Այսինքն մակ հասցեն լինում է վերջին ցանցի վերջին հոսթինը, որով անցնում է պակետը։ Ամեն անգամ մակ ադրեսը գցվում է ու տրամադրվում է նորը։ 
        swich֊ի մեջ աղյուսակ է գրված, որ port֊ը որ macaddres֊ին է ուղղարկելու, իր մեջ ուղղորդումը տեղի է ունենում մակ ադրեսսների մակարդակով։ Մինչև ip ստանալը հոսթը ինդենտիֆիկացվում է մակ ադրեսով և լոկալ ցանցում աշխատում է swich֊ը։ OSI data link level աշխատող սարք է։
router - պակետները ուղղորդելու համար նախատեսված գերարագագործ համակարգիչ է, իր միջոցով տվյալը ուղղորդվում է դեպի ճիշտ այփի։ Աշխատում է ցանցից ցանց կապ ապահովելիս։ Ունեն իրենց սեփական օպերացիոն համակարգը։ OSI network level աշխատող սարք է։


DHCP server (dynamic host configuration protocol) ֊ սերվեր է, որը ունի այփիների դիապազոն, որը կոչվում է ip pool, դիմում են սերվերին այդ դիապազոնից այփի ստանալու համար, սերվերը տրամադրում է։ Որպես կանոն DHCP server֊ները ադմինի համակարգչի վրա են։ Տրամադրվաշ այփին կարող է փոխվել ամեն անգամ, կամ կարող է ստատիկ լինել։
ARP (address resolution protocol) request - host֊ը հարցում է ուղղարկում ցանցի բոլոր անդամներին, այփի ստանալու համար, ստանում է նաև DHCP server֊ը ու որպես պատասխան տալիս է այփի հոսթին, այն իդենտիֆիկացնելով մակ հասցեի միջոցով։ 
DNS (Domian names server) -  ասոցացնում է կայքի անունը  ip֊ի հետ։ DNS կա բրաուզերի քեշի մեջ, համակարգչի մեջ։ Վերջիններիս մեջ չգտնելու դեպքում պրովայդերի DNS server-ին կդիմի։
web hosting - տվյալ սայթի ֆայլերը հյուրընկալվել են տվյալ սերվերի վրա, այլ կերպ ասած տրամադրվում է հիշողություն, որտեղ կարելի է տեղադրել կայքի ֆայլերը, սերվերը 24/7 ռեժիմում հասանելի է, մեր հիշողության հետ կապված մեզ տրամադրվում է այփի, որով կարող ենք հասանելիություն ունենալ ֆայլերին։
ping - բաշի հրաման է, instant message-ներ է ուղղարկում ինչ֊որ կայքի, դատարկ փեքիջներ են, ստուգելու համար կապը։ Ping կարելի է անել ip֊ով էլ, կայքի անունով էլ։
ifconfig, netstat - bash shell commands for internet checkings 
Provider - մեծ հաշվով ռաուտեռների համակարգ է։ 
Default getway - տրամադրում է պրովայդերը, երբ տվյալ այփիով հոսթը միանգամից չի գտնվում դիմում է պրովայդերի տրամադրած default ip֊ին, տվյալ ip֊ով արդեն հնարավոր է ճշտել փնտրվող հոսթը։ Ցանցին արաջին անգամ կպնելուց, հոսթը դիմում է default getway֊ին, որը տրամադրում է ցանցին կպած հոսթերի այփիները։ Իսկ նոր կպած հոսթի այփին բրոդքասթ ընդեսով տարածվում է ցանցում և գրանցվում ցանցի հոսթերի ռաութինգ թեյբլի մեջ։
Lookback ip - local host, տվյալ հոսթի հասցեն է
Network mask (subnet mask) - ip֊ին բաղկացած է ցանցի ու հոսթի մասից։ Այդ երկուսը ֆիքսված չափի չեն։ ip֊ին կարող ենք դիտարկել, որպես 0 և 1֊երի հաջորդականություն։ Երբ ռաուտերին ինչ֊որ հասցե տալիս ենք այդ հասցեի միայն ցանցի մասն է օգտագործվում երթուղավորումը կազմակերպելու համար։ Նեթվորկ մասկը օգտագործվում է ip֊ի միջից ցանցի մասը դուրս կորզելու համար։ ip֊ի հետ ռաուտերին է ուղղարկվում նաև մասկը, որը 1֊երի հաջորդականություն է ցանցի բիթերում, 0֊ների հաջորդականություն է պորտի մասում։ Network mask֊ը և ip֊ին իրար հետ & է արվում, այդպիսով դուրս է բերվում ցանցի մասը։
Broadcat address - ամեն ցանց ունի իր բրոադքասթ հասցեն, այս հասցեին ուղղարկված ինֆորմացիան գնում է տվյալ ցանցի բոլոր հոսթերին։ Տվյալ ցանցի ամենավերջին թիվը ստանդարտով հատկացրվում է այս հասցեի համար, եթե մենք այս հասցեին ուղարկենք ինֆորմացինա, կստանան ցանցի բոլոր հոսթերը։
Routing table - այս աղյուսակում պահվում է ցանցին կպած հոսթերի ip֊ների ցուցակը։
